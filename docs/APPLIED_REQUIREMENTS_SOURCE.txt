Sen Agent Mode’da çalışan kıdemli full-stack AI systems engineer, ML/NLP/CV pragmatisti, ürün mimarı ve teknik dokümantasyon sorumlusu gibi davranacaksın.

Bu görevin amacı mevcut “XNative Recursive Learning / Futbol Odaklı Yapay Zeka Destekli X İçerik Performans Sistemi” projesini baştan sona incelemek, bizim konuştuğumuz gerçek hedefe göre yeniden hizalamak, eksikleri tamamlamak, çalışan bir MVP hâline getirmek, optimizasyonları yapmak, testleri yazmak ve tüm belgeleri güncellemektir.

## 0. Mutlak hedef

Bu proje bir haber botu değildir.

Amaç:
Tek bir futbol odaklı X hesabı için X/Twitter’daki gerçek futbol kültürünü, kısa doğal dili, mizahı, tepki formatlarını, alıntı tweet davranışını, medya bağlamını ve performans geri bildirimini öğrenen; kullanıcıya paylaşım önerileri üreten; ancak ilk sürümde otomatik paylaşım yapmayan, insan onaylı, düşük maliyetli / ücretsiz çalışabilen bir içerik performans sistemidir.

Sistem “haber dili” üretmeye çalışmayacak. “Habere benzeyen kurumsal metin”, “yapay zeka gibi açıklayıcı paragraf”, “basın bülteni tonu”, “fazla düzgün ve resmi cümle” üretmeyecek. Sistem X’teki futbol hesabı dilini öğrenecek: kısa, doğal, bağlama uygun, bazen reaksiyon, bazen alıntı tweet, bazen mizahi, bazen nötr, bazen taraftar refleksine yakın ama risk kontrollü.

## 4. Veri toplama stratejisi

Ana kaynak X API olmayacak.

Öncelik sırası:

1. Browser extension / content script

   * Kullanıcının açtığı X sayfasındaki post kartlarını DOM’dan oku.
   * Post text, author handle, display name, timestamp, post URL, quote tweet text, quoted author, visible metrics, media URLs/preview, alt text varsa al.
   * Otomatik scroll agresif yapılmasın.
   * Kullanıcı capture başlatır veya düşük frekanslı izleme yapar.
   * Aynı post tekrar alınmasın.

2. Playwright persistent browser context

   * Kullanıcı kendi tarayıcı oturumunu yönetir.
   * Şifre/2FA hiçbir şekilde kod tarafından alınmaz.
   * Login gerekirse kullanıcı browser’da kendi girer.
   * Sistem sadece görünür sayfayı okur.
   * Playwright modu opsiyonel ve dikkatli kullanılmalı.

3. Manual import

   * Kullanıcı post URL, ekran görüntüsü, metin veya JSON fixture yükleyebilmeli.
   * Geliştirme/test için sample fixtures mutlaka olmalı.

4. RSS / haber / maç verisi

   * Sadece doğrulama ve bağlam.
   * Asıl dil/stil öğrenme kaynağı değildir.

Yasak:

* X API zorunlu kullanım.
* Ücretli scraper.
* Proxy rotation.
* Login credential capture.
* Otomatik paylaşım/follow/like.

## 6. NLP yaklaşımı

Overengineering yapma.

İlk sürüm NLP bileşenleri:

1. Text cleaning

   * URL temizleme
   * mention/hashtag koruma
   * emoji koruma
   * fazla boşluk
   * quote context ayrımı
   * Türkçe karakter koruma

2. Language detection

   * fastText varsa kullan.
   * Yoksa langdetect fallback.
   * Dil tespit hatası akışı kırmasın.

3. Embedding

   * Varsayılan: sentence-transformers ile küçük multilingual model.
   * Öneri: paraphrase-multilingual-MiniLM-L12-v2 veya all-MiniLM-L6-v2.
   * Model yoksa TF-IDF fallback.
   * Embedding cache zorunlu.
   * Aynı text tekrar embed edilmesin.

4. Style memory

   * Onaylanmış veya yüksek performanslı X postlarından örnek havuzu oluştur.
   * Yeni olay için benzer stil örneklerini getir.
   * Retrieval top_k: 5.
   * Benzerlik: cosine similarity.
   * Çok benzer/duplicate örnekleri filtrele.
   * Haber dili değil, X-native kısa futbol dili hedefle.

5. Tone analyzer

   * Kurallı etiketlerle başla:

     * reaction
     * dry_humor
     * fan_voice
     * neutral_short
     * quote_bait_without_clickbait
     * ironic
     * serious_risk
   * Sonra feedback ile ağırlıklar güncellensin.

6. Quality filter
   AI kokan ifadeleri işaretle:

   * “Bu bağlamda”
   * “Özetle”
   * “Dikkat çekici bir gelişme”
   * “Futbolseverler için önemli”
   * “Sosyal medyada gündem oldu”
   * “Kritik bir gelişme yaşandı”
   * “X platformunda”
   * “taraftarlar ikiye bölündü” gibi klişe ve haberci ifadeler
   * Çok açıklayıcı, uzun, resmi cümleler
   * Gereksiz noktalama ve yapay emoji

7. Draft generation

   * Önce template + style retrieval ile çalış.
   * LLM varsa sadece kalite artırma için opsiyonel.
   * LLM yoksa sistem yine 3 öneri üretsin.
   * Öneriler 280 karakter sınırına uygun olsun.
   * Her olay için en az 3 varyant:

     * kısa tepki
     * alıntı tweet tonu
     * nötr ama X dili
   * Risk yüksekse “paylaşma / bekle / doğrula” önerisi üret.

## 7. CV / medya yaklaşımı

İlk sürümde ağır CV yapma. Gerekli ve hafif bileşenleri uygula.

1. Medya dedup

   * pHash / imagehash
   * Complexity: her medya için O(1) hash; benzerlik için küçük arşivde O(n), büyürse LSH/VP-tree opsiyonel.
   * Aynı görsel tekrar işlenmesin.

2. OCR

   * Öncelik DOM text ve alt text.
   * OCR sadece:

     * görselde metin olabilir
     * alt text yok
     * post metni yetersiz
   * Tesseract / EasyOCR / PaddleOCR opsiyonel.
   * OCR yoksa akış kırılmasın.

3. Video

   * Tam video analizi yok.
   * İlk sürümde thumbnail veya 1-3 frame sample yeterli.
   * ffmpeg/opencv opsiyonel.
   * Ağır frame extraction yapma.

4. Image-text matching

   * CLIP/open_clip opsiyonel.
   * Eğer model kurulamazsa:

     * alt text
     * OCR text
     * filename/url hints
     * user-provided tags
     * pHash clusters
       ile çalış.
   * Ama mimaride CLIP opsiyonel plug-in olarak hazır olsun.

5. Medya risk

   * copyright unknown ise otomatik paylaşım önerisi riskli say.
   * Medya önerileri “kullan”, “kontrol et”, “kullanma” olarak işaretlenmeli.
   * Medya indirme sınırlı ve cache kontrollü olsun.
   * Sınırsız medya arşivleme yok.

6. Belalcaggins benzeri test

   * Bir hesap açıkça futbol hesabı olmasa bile futbol içeriği paylaşıyor olabilir.
   * Sadece keyword ile karar verme.
   * Şu sinyalleri birlikte kullan:

     * post text
     * quote tweet text
     * medya alt text
     * OCR text
     * görsel tag/CLIP tag
     * futbol hesaplarıyla etkileşim
     * kullanılan emoji / maç günü dili
     * benzer stil örnekleri
   * Sonuç: “kaynak aday puanı” ve “neden aday?” açıklaması ver.

## 8. Skor algoritmaları

Aşağıdaki algoritmaları uygula veya mevcutsa düzelt.

### 8.1 Event score

Amaç: olay paylaşmaya değer mi?

Önerilen formül:

event_score =
0.25 * source_reliability

* 0.20 * source_count_score
* 0.15 * velocity_score
* 0.15 * relevance_score
* 0.10 * novelty_score
* 0.10 * media_context_score

- 0.20 * risk_score
- 0.10 * fatigue_score

Normalize et: 0-100.

Açıklanabilir olmalı:

* hangi sinyal kaç puan verdi
* hangi risk puan kırdı
* neden önerildi / neden önerilmedi

### 8.2 Source reliability

Başlangıçta kurallı:

* official club: yüksek
* güvenilir gazeteci: yüksek
* taraftar hesabı: orta/düşük
* yeni aday: düşük/unknown
* geçmiş yanlış bilgi: ceza
* erken doğru sinyal: ödül
* yüksek etkileşim ama riskli dil: ayrı risk

Feedback ile güncelle:
new_score = old_score * 0.85 + observed_score * 0.15

### 8.3 Account candidate score

Aday takip hesabı için:

candidate_score =
0.25 * football_relevance

* 0.20 * early_signal_quality
* 0.15 * media_context_quality
* 0.15 * quote_context_quality
* 0.10 * language_style_fit
* 0.10 * engagement_quality

- 0.20 * misinformation_risk
- 0.10 * toxicity_risk

Sonuç:

* auto-follow yok
* “Aday olarak ekle”
* “Reddet”
* “1 hafta izle”
* açıklama ve örnek postlarla birlikte göster

### 8.4 Novelty / fatigue

Aynı konu çok tekrarlandıysa düşür.

* Text similarity > 0.85 ise duplicate/near-duplicate
* Aynı oyuncu/kulüp/olay 2 saat içinde çok tekrarlandıysa fatigue artır
* Aynı medya pHash benzerliği yüksekse tekrar say
* Çok sık aynı formatta öneri üretme

### 8.5 Style selection

Yeni olay için:

1. event_type belirle
2. risk seviyesini belirle
3. style memory’den benzer 5 örnek getir
4. başarılı tonları ağırlıklandır
5. 3 varyant üret
6. quality filter’dan geçir
7. risk filter’dan geçir
8. onay kuyruğuna ekle

### 8.6 Learning

İlk sürümde model fine-tuning yapma.

Online learning:

* Kullanıcı onayladıysa ilgili ton, kaynak, medya tipi, event_type ağırlığını artır.
* Reddettiyse azalt.
* Revize ettiyse edited_text’i style_memory’ye aday olarak ekle.
* Postlandıktan sonra performans metrikleri girilirse ödül sinyali üret.

Önerilen basit update:
weight_new = clamp(weight_old + learning_rate * reward * feature_value, min, max)

Learning rate: 0.03-0.10 arası.

Contextual bandit opsiyonel:

* Thompson Sampling veya epsilon-greedy.
* İlk sürümde epsilon-greedy yeterli.
* Amaç: ton/format/media kombinasyonlarını kontrollü denemek.

## 9. Performans ve optimizasyon

Zorunlu optimizasyonlar:

1. Cache

   * post dedup hash
   * media pHash
   * OCR result cache
   * embedding cache
   * LLM response cache, varsa
   * source profile cache

2. Complexity hedefleri

   * N post ingest: O(N)
   * dedup lookup: O(1)
   * keyword/rule scoring: O(N*K), K küçük kalmalı
   * embedding search küçük veri için O(N*d), büyürse sklearn NearestNeighbors / FAISS opsiyonel
   * media pHash lookup küçük veri için O(M), büyürse bucket/LSH
   * dashboard query pagination zorunlu

3. Async / worker

   * UI thread bloklanmasın.
   * Capture hızlı dönsün.
   * OCR/embedding/LLM gibi pahalı işler background worker queue’ya alınsın.
   * İlk sürümde Redis/Celery zorunlu değil; SQLite job queue veya APScheduler yeterli.

4. Rate limiting

   * Browser capture düşük frekanslı olsun.
   * Aynı sayfa tekrar tekrar parse edilmesin.
   * Circuit breaker: X sayfası parse hatası artarsa dur.

5. Memory

   * Büyük medya dosyalarını sınırsız tutma.
   * Thumbnail ve metadata öncelikli.
   * Video frame sayısı sınırlı.
   * Embedding model lazy-load.

6. Docker

   * docker compose up ile çalışmalı.
   * En az şu servisler:

     * api
     * ui
     * worker
     * db veya sqlite volume
   * .env.example olmalı.
   * Local volume: data/, media_cache/, logs/.
   * API key olmadan sample mode çalışmalı.

## 10. Error handling ve logging

Her modülde şu standartları uygula:

* typed exceptions
* retry + exponential backoff
* timeout
* graceful fallback
* structured logging
* no silent fail
* user-facing error message
* audit log

Örnek:

* OCR yoksa: “OCR unavailable, used alt_text/text fallback.”
* embedding model yoksa: “Embedding unavailable, used TF-IDF fallback.”
* LLM yoksa: “LLM unavailable, used template fallback.”
* X page parse bozulursa: “DOM parser failed, screenshot/manual import required.”
* DB locked olursa retry.
* Duplicate post ise ingest etme ama logla.
